## v1.0.0 — Initial Release

Build Flutter mobile UIs entirely from PHP. Your app lives in `app/MobileUI/` — never touch Flutter except to build and publish.

### How it works
PHP (app/MobileUI/) → JSON manifest → Flutter renderer → Native app

### Included
- **33 UI Components** — Scaffold, Column, Row, Text, Button, Card, RemoteList, StoreList, Clickable, Image, Carousel, Modal, Badge, Loader, InfiniteScrollList, DynamicForm, DynamicTable, GridView, BottomSheet, Switch, AuthGuard, When, SearchBar, SectionHeader, Dropdown, TextInput, EmptyState, ErrorView, Spacer, Padding, Icon, RemoteDetail, DynamicCard
- **25 Actions** — navigate, apiCall, setStoreValue, clearStore, addToCart, showModal, showNotification, goBack, native, registerPushToken, pusherConnect, socketConnect, openCamera, more
- **State System** — Reactive stores with computed fields, bindings via `{{store.name.field}}` and `{{item.field}}`
- **Auth** — Sanctum integration, `resultMap`, guest redirect, secure token storage
- **CLI Commands**
  - `flutter-php:generate-structure` — Generate JSON manifest
  - `flutter-php:init-app` — Scaffold new app
  - `flutter-php:make-screen` — Create screen
  - `flutter-php:make-component` — Create component
  - `flutter-php:make-store` — Create store
- **Pagination** — `InfiniteScrollList` with scroll-to-load
- **Forms** — `DynamicForm` with validation and submit action
- **Tables** — `DynamicTable` with DataTable rendering
- **Modals** — `showModal` action opens bottom sheet with custom content
- **Cart** — `addToCart`, `removeFromCart`, `updateCartQuantity` with computed totals
- **Persistence** — `persist: true` on any store saves to local storage

### Install
```bash
composer require prolaxu/flutter-php
```

### Quick Start
```bash
php artisan flutter-php:init-app MyApp
php artisan flutter-php:generate-structure
```

Load `GET /api/app-structure` in your Flutter renderer.

### Requirements
- PHP 8.2+
- Laravel 10/11/12/13
